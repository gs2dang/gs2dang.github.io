위즈위그 에디터란?

위지위그(WYSIWYG)는 What You See Is What You Get 준말이다. 사용자가 문서를 작성할 때 입력한 문자, 표, 이미지 등의 출력물들이 화면에 그대로 나오게 해주는 것을 말한다.

출처: 위키피디아


시작

여러 개 에디터 중에서 Django-CKEditor를 사용해 에디터를 구현한다.

# forms.py
from ckeditor.fields import CKEditorWidget

class PostForm(forms.ModelForm):
    class Meta:
        model = Post
        fields = ['title', 'content']
        widgets = {
            'title': forms.TextInput(attrs={'class': 'form-control'}),
            # 기존 코드 forms.Textarea(attrs={'class': 'form-control'})
            'content': forms.CharField(widget=CKEditorWidget()),
        }


위지위그 적용이 안 됨

글쓰기에서는 표, 글씨 굵기, 기울기 등 모든 게 잘 적용됐다. 하지만 실제 게시글 모습은 바로 위 사진처럼 에디터가 적용되지 않은 채 출력됐다. 다행히 스택 오버플로에서 otaviosoares 답변으로 쉽게 해결할 수 있었다.

여기까지 하면 문자 꾸미기나 표 입력을 할 수 있는 위지위그 에디터를 사용할 수 있고, 해당 글에서도 잘 나오는 것을 확인할 수 있다. 이제는 이미지 업로드도 할 수 있도록 해보자.

# forms.py
from ckeditor_uploader.fields import RichTextUploadingField

class PostForm(forms.ModelForm):
    class Meta:
        model = Post
        fields = ['title', 'content']
        widgets = {
            'title': forms.TextInput(attrs={'class': 'form-control'}),

            'content': forms.CharField(widget=RichTextUploadingField()),

RichTextField 에서 RichTextUploadingField 수정했다. 이렇게 하면 인터넷 이미지 URL 또는 내 컴퓨터에 있는 이미지를 업로드하여 보여줄 수 있다. 이미지를 업로드하기 위해 파일을 선택하고 서버 전송을 누른다. 하지만 이미지는 나타나지 않는다. 폴더에 이미지가 저장되는 것을 확인했는데, 왜 화면에 표시되지 않을까?

그건 .static 파일과 다르게 개발서버(DEBUG=True)에서는 기본 서빙을 지원하지 않는다. 그래서 개발 편의를 위해 아래의 코드를 추가를 해주면 이제는 이미지가 제대로 나오는 것을 확인할 수 있다.

Admin 유저만 되는 문제 및 해결

관리자 계정으로는 이미지를 업로드가 잘 되고 화면에도 잘 표시가 됐다. 하지만 관리자가 아닌 유저가 이미지 업로드를 하면 잘못된 서버 응답 에러가 발생하면서 업로드가 안 됐다. django ckeditor incorrect server response 검색하고 찾는 중에 스택 오버플로 Mike Robinson 답변을 찾았다. 이 답변자 말에 따르면 기본 urlpatterns는 스태프 전용이기에 에러가 뜬 거라고 답했다. 그리고 아래 코드의 해결 방법을 제시했다. 이로써 일반 유저도 이미지를 업로드할 수 있게 되었다.

1
2

# urls.py
from ckeditor_uploader import views as uploader_views

urlpatterns = [
    ...
    path('ckeditor/upload/', uploader_views.upload, name='ckeditor_upload'),
    path('ckeditor/browse/', never_cache(uploader_views.browse), name='ckeditor_browse'),
]

Note that by adding those URLs you add views that can upload and browse through uploaded images. Since django-ckeditor 4.4.6, those views are decorated using @staff_member_required. If you want a different permission decorator (login_required, user_passes_test etc.) then add views defined in ckeditor.urls manually to your urls.py.

도커에서 StaticFiles 불러오지 못할 때

로컬 환경에서는 잘 됐다 CSS 적용도 잘 되고 위지위그 에디터도 잘 됐다. 하지만 도커에서는 모든 게 안 됐다. 혹시나 해서 도커를 배포했지만 증상은 똑같았다. 개발자 도구에서는 ERR_ABORTED 403(Forbidden) 에러가 표시됐다. 문제를 해결하기 위해서 구글에 검색했다. 그리고 Install CKEditor in django v2.1 with youtube and code spinet plugin 글을 찾았다. 이 글 6번에 S3 설정하는 방법이 나오는데 아래와 같은 코드가 있었다. 입력하고 도커 빌드 후 확인했다. 잘 된다. 그리고 배포 환경에서도 잘 된다.

기존에 설정한 STATIC_URL = ‘/static/’ 와 무엇이 다른지 그리고 왜 이렇게 설정했는지 알아보자.

2

AWS_S3_CUSTOM_DOMAIN = '%s.s3.amazonaws.com' % AWS_STORAGE_BUCKET_NAME
AWS_LOCATION = 'static'
STATIC_URL = 'https://%s/%s/' % (AWS_S3_CUSTOM_DOMAIN, AWS_LOCATION)
