1. 블로그 시작

정리한 내용을 마크다운에 작성 후 내 컴퓨터에 저장하거나 에버노트 또는 노션과 같은 서비스에 정리할 수 있다. 하지만 이렇게 하면 나만 본다고 대충 정리하게 되고 나중에는 “음… 왜 이렇게 적혀있지?“처럼 내가 정리한 것도 잘 몰라보는 경우가 있다. 그래서 보기 쉬운 글을 적기 위해 블로그를 시작하기로 했다.

1.1 Hugo를 선택한 이유

큰 이유는 없다. 이전에 잠깐 사용한 Jekyll 블로그는 Ruby 의존성 패키지, 버전 관리 등 설치해야 할 것이 너무 많다. 그게 너무 귀찮아서 Hugo를 선택하게 됐다.

#!/bin/bash

echo -e "\033[0;32mDeploying updates to GitHub...\033[0m"

# Build the project.
hugo -t <theme_name>

# Go To Public folder
cd public

# Add changes to git.
git add .

# Commit changes.
msg="rebuilding site `date`"
if [ $# -eq 1 ]
  then msg="$1"
fi
git commit -m "$msg"

# Push source and build repos.
git push origin master

# Come Back up to the Project Root
cd ..

# blog 저장소 Commit & Push
git add .

msg="rebuilding site `date`"
if [ $# -eq 1 ]
  then msg="$1"
fi
git commit -m "$msg"

git push origin master

위 내용을 입력 후 저장하면 다음에는 아래 명령어를 입력하자.

1

❯ chmod +x deploy.sh

이제 ./deploy.sh 명령어를 입력하면 아까처럼 번거로운 작업 없이 배포를 할 수 있다.
