이전 글에서는 간단히 Heroku에 Django를 배포하는 거라면 이번에는 settings.py 분리, 비밀 키 분리 등 내용을 좀 더 추가해서 배포를 한다.


디렉터리 구조

1

3
4

. << 최상위 레벨: 저장소 루트
└── src << 두 번째 레벨: 프로젝트 루트
    ├── config << 세 번재 레벨: 설정 루트
    │   ├── __init__.py
    │   ├── settings.py
    │   ├── urls.py
    │   └── wsgi.py
    └── manage.py
  • 최상위 레벨은 프로젝트 폴더다.
  • 두 번째 레벨은 django-admin startproject로 만들어진 폴더다.
  • Github 저장소를 이용하게 된다면 최상위 레벨에서 git init을 하고 .gitignore, requirements.txt 파일들을 이곳에 위치했겠지만, Heroku 저장소는 두 번째 레벨을 포함해 그 밑에 있는 파일들을 올린다. 뒤에 나오는 Procfile, runtime.txt, .gitignore, requirements.txt 등 파일들은 이 위치에 있어야 한다.


Django 설정

1. settings.py 분리

settings.py를 패키지로 만들어서 환경에 맞는 설정 파일을 추가한다. 분리하면 __init__에 있는 코드들은 base.py로 옮긴다. 공통되는 설정은 base.py에 놔두고, 그 외는 해당 설정 파일에 작성한다.

    __init__.py # 비어있는 파일
    base.py # 공통되는 설정
    local.py # 개발 환경
    production.py # 운영 환경

1-1. BASE_DIR 경로

1
2

# 기존
BASE_DIR = #1 os.path.dirname(#2 os.path.dirname(os.path.abspath(__file__)))

기존 BASE_DIR 설정대로라면 #2 위치는 settings 디렉터리(네 번째 레벨), #1 위치는 config 디렉터리(세 번째 레벨)가 된다. os.path.dirname을 하나 더 추가해 BASE_DIR 변수가 src 디렉터리(두 번째 레벨)가 되도록 한다.

1
2

# 수정
BASE_DIR = os.path.dirname(os.path.dirname(os.path.dirname(os.path.abspath(__file__))))

DEBUG 설정은 개발 환경에서는 True로 설정하나, 운영 환경에서는 False로 설정한다. 환경에 따라 DB, ALLOWED_HOSTS 등 설정 값이 다르기에 각 환경에 맞게 해당 파일에 설정을 적어주면 된다.

3. SECRET_KEY 분리

상대방이 내 비밀 값들을 볼 수 없도록 해야한다. 만약 노출이 된다면 누구든지 내 코드에 접근할 수 있게 된다. 로컬 환경에서는 환경 변수 또는 JSON 방식으로 비밀 키를 참조하고, 운영 환경에서는 Heroku에 설정을 추가한다.


3-1. 환경 변수

위키피디아에서는 환경 변수를 ‘프로세스가 컴퓨터에서 동작하는 방식에 영향을 미치는, 동적인 값들의 모임이다’라고 설명한다. 무슨 말인지 잘 모르겠다…

환경 변수를 설정하려면 bashrc, .bash_profile 또는 .profile의 뒷부분에 추가하면 된다. bash가 아닌 zsh를 사용하기 때문에 .zshrc 파일에 SECRET_KEY를 입력한다.

1
2
3
4

$ vim ~/.zshrc

# .zshrc 맨 밑에 export 키="값" 형태로 추가한다.
export TUTORIAL_SECRET_KEY="비밀 키 입력"

터미널에서 파이썬을 열어 아래의 코드를 입력하면 정상적으로 값이 출력되는 것을 확인할 수 있다.

1
2

>> import os
>> os.environ["TUTORIAL_SECRET_KEY"]

그리고 local.py 파일에 아래 내용을 적는다.

1
2
3

# settings/base.txt

SECRET_KEY = os.environ["TUTORIAL_SECRET_KEY"]

만약 runserver를 했는데 KeyError가 발생한다면 실행 중인 IDE를 재시작 한다.


3-2. JSON 파일

여기서는 환경 변수 방법 대신에 JSON 방식을 사용한다. 프로젝트 루트에 .secrets 폴더를 만든다.

    ├── config

2) JSON 파일 만들기

이 폴더 안에 base.json 파일을 만들고 아래의 내용을 추가한다.

    "SECRET_KEY":"비밀 키"

3) settings 파일에 추가

아래의 내용들을 추가한다.

1
2
3

# settings/base.py

SECRET_DIR = os.path.join(BASE_DIR, '.secrets')
1
2
3
4
5

# settings/local.py
import json

SECRETS = json.load(open(os.path.join(SECRET_DIR, 'base.json')))
SECRET_KEY = SECRETS['SECRET_KEY']
  1. 파일을 먼저 열어야 한다. “.secrets’ 폴더에 있는 base.json 파일을 불러온다.
  2. 파일 객체를 json.load를 이용해 파이썬 객체로 만들어 주는 디코딩을 한다.
  3. SECRETS 변수는 {"SECRET_KEY": "비밀 키"} 가 저장되어 있다.
  4. 딕셔너리 값을 가져오는 방식을 이용해 SECRET_KEY 변수에 저장한다.

4) .gitignore 추가하기

분리한 이유는 상대방이 내 비밀 키를 볼 수 없도록 하기 위해서다. 그래서 이 파일 또한 상대방에게 보여주지 않기 위해 .gitignore에 추가하여 커밋에 제외시킨다.

1
2
3

# .gitignore

.secrets/


3-3. 운영 환경

git push를 할 때 비밀 값들이 포함된 파일들은 제외되기 때문에 Heroku에서 알 수 있도록 다른 설정을 해줘야 한다. 여기서는 heroku config:set 또는 heroku 대시보드를 이용해 설정을 저장한다.

Heroku에 설정을 저장하기 위해 먼저 Heroku에 로그인을 하고 앱을 만들어 준다. <app-name>.herokuapp.com 주소를 사용하기에 app-name은 다른 사람이 사용하지 않는 이름으로 지어야 한다.

1
2
3

$ heroku login

$ heroku create <app-name>

1) Heroku CLI를 이용해 설정 저장

1

$ heroku config:set SECRET_KEY='비밀 키'

큰따옴표가 아닌 작은따옴표 안에 내용을 넣어야 한다. 그러면 해당 앱 Settings에서 설정이 저장된다.

2) Heroku 대시보드

헤로쿠 대시보드에서 Settings으로 이동하면 Config Vars 부분이 보인다. Revel Config Vars를 눌러 키와 값을 입력할 수 있다. 두 곳 모두 따옴표 없이 그냥 입력한다.


4. requirements.txt

Django처럼 어느 환경에서나 공통으로 이용하는 것이 있는 반면 django-debug-toolbar와 같은 개발 환경에서만 이용하는 경우가 있다. 이렇게 환경에 따라 설치하는 것이 다르다. 그래서 pip freeze > requirements.txt 명령어를 입력하기 보다는 pip list를 입력해 환경에 맞게 나누고 적는 것이 좋다.

프로젝트 루트에 여러 파일을 만들기보다는 requirements 폴더를 만들고 이 안에 파일을 만드는 것이 보기도 좋고 관리하기도 쉽다. Heroku 저장소에 올린다면 requirements.txt 파일은 프로젝트 루트(manage.py가 있는 위치)에 있어야 한다. 이 파일을 requirements/production.txt로 연결시켜주자.

5. DB

Django 기본 데이터 베이스는 sqlite3를 사용한다. 하지만 Heroku에서는 sqlite3를 사용하지 않고 PostgreSQL을 사용한다. 로컬 환경에서 sqlite3는 사용해도 되긴 하지만 두 환경 모두 같은 데이터베이스 사용하는 것을 권장한다.


5-1. 로컬 환경

1) PostgreSQL install

1
2

$ sudo apt update
$ sudo apt install postgresql postgresql-contrib

2) PostgreSQL 들어가기

1
2

$ sudo -u postgres psql
$ sudo -u postgres psql 데이터베이스_이름

3) DATABASE 생성

1
2
3

$ sudo -u postgres createdb 데이터베이스_이름
또는
postgres=# CREATE DATABASE db_name;
  • sudo -u postgres psql 들어가면 postgres=# 가 나타난다.
  • db_name은 Django DB settings에서 NAME에 해당된다.

4) User 생성

1

postgres=# CREATE USER user_name WITH PASSWORD 'user_password';
  • ‘user_password’ 적을 때, 작은따옴표(’ ‘) 안에 적어줘야 한다.
  • user_name은 Django DB settings에서 USER에 해당된다.
  • user_password는 Django DB settings에서 PASSWORD에 해당된다.

5) 인코딩, isolation, timezone 설정

postgres=# ALTER ROLE user_name SET default_transaction_isolation TO 'read committed';
postgres=# ALTER ROLE user_name SET timezone TO 'UTC';

6) DB 접근 권한

1

postgres=# GRANT ALL PRIVILEGES ON DATABASE db_name TO db_user;

5-2. 운영 환경

Heroku 대시보드에서도 만들 수 있지만 여기서는 CLI를 이용해 만든다. 먼저 django-heroku 패키지를 설치한다. DB 관련 패키지와 정적 파일 관련 패키지가 설치된다. 패키지를 설치하면 pip list를 참조하여 requirements/production.txt 에 입력한다.

1

$ pip install django-heroku
1
2
3

$ vim requrements/production.txt

django-heroku==0.3.1

1

$ heroku addons:create heroku-postgresql:<PLAN_NAME> --version=9.5

git init을 먼저 했다면 Heroku 앱을 만들 때 자동으로 Heroku 저장소로 연결되어 아래와 같은 에러는 보지 않을 것이다. 하지만 그게 아니라면 에러를 보게 될 것이다. 해결법에는 두 가지가 있다.

1) –app app-name 추가

기존 명령어 뒤에 -a[--app] <app-name>를 추가한 명령어를 입력한다.

2) git init

1
2

$ heroku addons:create heroku-postgresql:hobby-dev

git init은 두 번째 레벨에서 해준다. 그리고 Heroku 저장소와 연결시켜준다. 그러면 -a <app-name> 없이도 DB 생성이 가능하다.

postgresql을 만들게 되면 Heroku 대시보드 Settings/Config Vars에 DATABASE_URL키와 값이 생성되는 것을 확인할 수 있다. 그리고 heroku addons 명령어를 다시 입력하면 No add-ons 대신 heroku-postgresql가 있다는 것도 볼 수 있다.

운영 환경 설정 파일에서는 DATABASES 설정을 {}로만 해준다. 기존 DB 설정 그대로 놔두어도 된다. dj_database_url을 사용해 Heroku의 DB를 사용할 것이기 때문이다. DB를 만들면 내 앱의 Resources 탭에서 확인할 수 있다.

1
2
3
4
5

# settings/production.py
import dj_database_url

DATABASES = {}
DATABASES['default'] = dj_database_url.config(conn_max_age=600, ssl_require=True)

DB를 클릭해 들어가면 아래 화면의 Overview로 가게 된다. 여기서 Settings/Database Credentials로 들어가면 Heroku CLI가 있는데, 이 명령어를 터미널에 입력하면 DB에 접속할 수 있다.
